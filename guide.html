User guide

SOIL CRC Benefit-Cost Analysis Tool Guide

This guide explains what each tab does, how the formulas are used, what each metric means, and how to interpret the results. It is written so the tool can be used without needing someone else to explain it.

1. What the tool does

The tool compares the financial performance of trial treatments using direct benefits and direct costs reported in the uploaded workbook. It converts yields into annual gross benefits using the grain price entered in the Settings tab, sums the direct cost components for each replicate row, discounts future annual benefits and annual costs over the analysis period, and reports the results using net present value, benefit-cost ratio, gross profit margin, return on investment, and the difference in net present value from the control treatment.

The tool is designed around the official Excel template. Each replicate must be entered on a separate row in the Trial_Data sheet. The results are most reliable when treatment names and treatment IDs are entered consistently across replicate rows.

2. How each tab works

Introduction gives the purpose of the tool, acknowledgements, and the main workflow.

Data is where you download the official template, upload the completed workbook, load example data, review validation messages, and enter the project details that appear at the beginning of the report.

Settings is where you enter the grain price, analysis period, discounting mode, initial discount rate, later discount rate, and switch year. Clicking Apply settings updates the schedule used in the calculations.

Results is where you run the analysis, compare a selected treatment against the control or compare two treatments side by side, review the ranking table, and inspect the graphs.

Sensitivity lets you test alternative discount rates, grain prices, benefit changes, and cost changes for the selected treatment.

Report brings the project details, analysis tables, graphs, and the written summary together. The report can be downloaded as a Word-compatible document or opened in a PDF-ready layout.

Help provides quick answers to common questions.

3. Data entry rules

Use one row per replicate. Keep the control treatment identified clearly as the control treatment. Keep treatment IDs and amendment names consistent across replicates for the same treatment. Yield and cost values should be numeric. The tool reads the cost components directly from the workbook and does not rely on a separate manually entered total cost value for analysis.

The report section in the Data tab is also important. The fields for the project name, collaborators, funding agency, project summary, and methodology description are carried through into the report and help explain the context of the trial and the meaning of the results.

4. Cost formula used by the tool

For each replicate row, direct costs are calculated as the sum of the workbook cost components used by the tool:

Direct cost per replicate = Seed cost + Total labour + Total machinery + Fertiliser + Pesticides + Fuel + Marketing + Repairs and maintenance + Support services + Electricity + Water + Insurance services + Overheads + Interest paid on loans + Structure maintenance + Other costs.

The tool then calculates the average direct cost for each treatment by averaging the replicate-level direct costs across all rows belonging to that treatment.

5. Discounting formulas

The tool assumes that the annual benefit and annual direct cost repeat each year over the chosen analysis period.

Annual benefit per hectare = Average yield × Grain price

Present value factor = Σ 1 / (1 + rt)t, where rt is the discount rate in year t.

Present value of benefits = Annual benefit × Present value factor

Present value of costs = Average direct cost × Present value factor

Net present value = Present value of benefits − Present value of costs

6. Output formulas

Benefit-cost ratio = Present value of benefits / Present value of costs

Gross profit margin (%) = (Present value of benefits − Present value of costs) / Present value of benefits × 100

Return on investment (%) = (Present value of benefits − Present value of costs) / Present value of costs × 100

Difference versus control = Treatment NPV − Control NPV

A higher NPV is better. A BCR above 1 means discounted benefits are larger than discounted costs. A higher gross profit margin means a larger share of discounted benefits remains after costs. A higher ROI means a larger gain relative to discounted costs.

7. Detailed numerical example using the included trial data

The example workbook included with the tool is based on the uploaded trial dataset. Under the default settings of a grain price of $500 per tonne, an analysis period of 10 years, and a constant discount rate of 5%, the discount factor used by the tool is:

Present value factor = 1/1.05 + 1/1.05² + ... + 1/1.05¹⁰ ≈ 7.7217

Using the treatment T11 Manure from the example data:

Average yield = 3.3475 t/ha

Average direct cost = $813.97/ha

Annual benefit = 3.3475 × 500 = $1,673.75/ha

Present value of benefits = 1,673.75 × 7.7217 ≈ $12,924.25/ha

Present value of costs = 813.97 × 7.7217 ≈ $6,285.24/ha

Net present value = 12,924.25 − 6,285.24 ≈ $6,639.01/ha

Benefit-cost ratio = 12,924.25 / 6,285.24 ≈ 2.06

Gross profit margin = (12,924.25 − 6,285.24) / 12,924.25 × 100 ≈ 51.4%

Return on investment = (12,924.25 − 6,285.24) / 6,285.24 × 100 ≈ 105.6%

For the control treatment T00 Control, the example data produce an NPV of about $4,094.41/ha under the same settings. Therefore, the NPV difference from control for T11 Manure is:

Difference versus control = 6,639.01 − 4,094.41 ≈ $2,544.60/ha

This means that, under the default assumptions used by the tool, T11 Manure produces about $2,544.60/ha more discounted net value than the control treatment over the 10-year analysis period.

8. How to read the ranking table

The ranking table sorts treatments from the highest NPV to the lowest NPV. This means the top treatment is the treatment with the strongest discounted net return under the assumptions currently entered into the tool. The ranking table also reports average yield, average direct costs, BCR, gross profit margin, ROI, and the NPV difference from the control so the user can see whether a treatment performs well because of higher yields, lower costs, or both.

9. How to read the graphs

The BCR graph shows how strongly discounted benefits exceed discounted costs. The NPV graph shows overall discounted net value. The NPV difference from control graph shows how much better or worse each treatment performs relative to the control treatment. The gross profit margin graph shows the percentage of discounted benefits remaining after costs. The ROI graph shows net gain relative to discounted costs. Together, these graphs help users compare treatments quickly.

10. Sensitivity analysis

The Sensitivity tab lets you test alternative assumptions without changing the original uploaded workbook. You can change the discount rate, grain price, benefit level, and cost level. This helps answer questions such as: What happens if prices fall? What happens if costs increase? What happens if the trial benefit is smaller than expected? The sensitivity table recalculates the key outputs under each scenario for the selected treatment.

11. Reporting workflow

The Report tab combines two parts. Part A reproduces the project information entered in the Data tab. Part B combines the ranking table, sensitivity results, graphs, and a narrative summary. If you have access to ChatGPT or Copilot, the tool prepares a prompt and opens it in a new window. You can then paste the resulting text back into the tool so it becomes part of the report. If you do not have access, the tool can generate a local draft summary that can be edited before exporting.

The Word report download creates a Word-compatible document. The PDF-ready option opens a print-friendly version of the report so it can be saved as a PDF from the browser print dialog.

12. Common problems and what to do

If the workbook cannot be read, make sure you are using the official template and that the file was saved before uploading. If results are blank, check that yield values and the cost fields are numeric and that the control treatment is present. If the ranking looks unexpected, review the grain price, analysis period, and discount settings. If the report preview is blank, run the analysis first and then refresh the report preview.
